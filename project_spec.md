# Ben 財務與投資模擬器｜Project Spec

> 版本：v1.0  
> 最後更新：2026-04-24  
> 專案定位：**個人財務狀況追蹤 + 通往致富的人生地圖 + 股票配置可視化與驗證系統**

---

## 0. 專案一句話

這個系統不是單純的股票回測工具，而是 Ben 的「人生財務作戰儀表板」。

它要同時回答三件事：

1. **現在到底有多少資產、負債、現金流與風險？**
2. **在合理安全的前提下，未來每個人生階段可能到達多少資產？**
3. **股票配置、現金水位、DCA、妥妥租收入、教育費與房貸槓桿，如何共同影響 Ben 的 long-term win rate？**

---

## 1. 核心專案目標

### 1.1 個人財務狀況追蹤

系統需追蹤 Ben 的完整財務狀態，而不是只追蹤美股。

必須包含：

- 美股資產
- 台股資產
- 台幣現金
- 外幣現金
- 房產市值
- 房貸負債
- 信用卡 / 帳單負債
- 保單資料
- Moneybook 記帳資料
- 妥妥租預測收入
- 薪資收入
- 租金收入
- 家庭生活費
- 教育費高峰
- 動態 DCA 投資水位

### 1.2 通往致富的人生地圖

本系統不再以「2034 年達成 1.45 億」作為硬目標。

應改為：

- 人生階段合理資產區間
- 達標機率
- 風險承受能力
- 中途出局機率
- 現金流安全性
- 教育費高峰承受度
- 退休 / 半退休可能性

系統應輸出不同年齡階段的合理目標：

| 年齡 | 階段 | 重點 |
|---:|---|---|
| 45 | 系統建立期 | 建立資料、分類、儀表板、DCA 規則 |
| 50 | 第一個財務自由檢查點 | 確認妥妥租衰退後是否仍安全 |
| 55 | 半退休 / 降低工作依賴檢查點 | 教育費高峰後，評估是否降工作強度 |
| 60 | 資產防守轉換期 | 從成長逐步轉向防守與現金流 |
| 65 | 高度財務自由期 | 管理退休、傳承、稅務與長壽風險 |

### 1.3 股票配比可視化與驗證

系統要讓 Ben 用數據比較不同配置，而不是靠感覺。

至少需比較：

- Current：真實現況
- Recommended：成長整理版
- Custom：防守版
- Candidate：Ben 成長防守平衡候選組合
- 100% VOO Benchmark：大盤基準
- VOO + BRK.B Benchmark：穩健核心基準

重點指標：

- P50 終值
- P5 worst case
- 最大回撤
- 資產耗盡機率
- 教育費高峰現金缺口
- 現金安全月數
- AI / 半導體曝險
- ETF 與個股重疊曝險
- 同跌風險
- 目標達成率

## 2. 回答原則與系統設計哲學

### 2.1 不追求最高紙上報酬

核心目標不是最大化模型終值，而是：

- 在安全合理範圍內提高長期報酬
- 提高 long-term win rate
- 降低中途出局機率
- 降低教育費高峰時被迫賣股的機率
- 降低房貸 / 利率 / 事業衰退帶來的現金流壓力

### 2.2 不把規則引擎當真理

所有模型輸出都必須標註：

- 已驗證事實
- 模型推論
- 人工假設
- 缺資料項目
- 可信度

系統不能只顯示結果，必須能解釋：

- 數字從哪裡來
- 哪個假設影響最大
- 哪些地方仍需人工確認
- 哪些結果只是方向性參考

### 2.3 不直接外推歷史高報酬

個股歷史績效不能直接等於未來報酬。

尤其 AI / 半導體 / 高成長股，需要使用：

- Base Case
- Bull Case
- Bear Case
- Monte Carlo
- 估值正常化情境
- 先殺估值再恢復情境

## 4. 收入規則

妥妥租收入 **以 Excel / CSV 預測表為主**。

請忽略先前口述固定值：

- 300 萬 / 年淨利
- 30 萬 / 月現金流

上述只能當歷史對話背景，不應作為模型預設。

### 4.2 妥妥租團隊認列規則

預測表中需依團隊認列：

| 團隊 | Ben 可認列比例 |
|---|---:|

模型必須輸出 audit table：

| year | team | raw_net_profit | recognition_rate | recognized_profit |
|---:|---|---:|---:|---:|

年度總認列：

```text
Ben 可認列妥妥租收入 = Σ(raw_net_profit × recognition_rate)
```

## 5. 支出規則

### 5.1 目前低消

目前家庭 / 個人 / 房貸 / 生活費等基本支出，以：

```text
2,000,000 TWD / 年
```

作為目前低消基準。

此 200 萬 / 年只視為目前低消，不含 2034–2038 教育費高峰。

### 5.2 Moneybook 支出資料品質

Moneybook 約從 2025/07 開始較可靠。

目前資料問題：

- 公司消費與個人消費混雜
- 妥妥租 / QLab / 公司修繕 / 代墊款可能混入個人消費
- 信用卡繳款可能造成重複計算
- 帳戶互轉不應計入收入或支出
- 投資買賣不是生活支出
- 稅金、醫療、旅遊需獨立標記為特殊支出

系統不能直接把 Moneybook 原始支出當作個人生活費。

### 5.3 Moneybook 清理規則初版

預設排除：

- 信用卡繳款
- 帳戶互轉
- 跨行轉帳
- 投資買賣
- 證券交易
- 妥妥租相關支出
- QLab / 學問相關支出
- 公司支出
- 租客 / 屋主 / 修繕 / 代墊

建議下一版改成可編輯規則表：

| keyword | rule_type | include_in_personal_expense |
|---|---|---|
| 妥妥租 | business | false |
| QLab | business | false |
| 信用卡費 | transfer | false |
| 醫療 | personal_special | true |
| 旅遊 | personal_quality | true |

## 8. 現金水位與 DCA 規則

### 8.1 現金安全水位

硬安全水位：

```text
1,500,000 TWD
```

這筆資金不投入市場。

用途：

- 支撐房貸利息
- 市場大跌時避免被迫賣股
- 家庭緊急預備金
- 事業波動緩衝

### 8.2 可投資現金

```text
可投資現金 = 現金 - 1,500,000
```

以現金 400 萬估算：

```text
可投資現金 = 4,000,000 - 1,500,000 = 2,500,000
```

### 8.3 動態 DCA

不應固定寫死每月 10 萬。

建議公式：

```text
每月可投資金額 = max(
  0,
  月收入
  - 月固定支出
  - 安全水位補足額
  - 未來 12 個月特殊支出準備
)
```

系統輸出三層：

| 模式 | 規則 |
|---|---|
| 保守 DCA | min(100,000, 可投資現金流 × 50%) |
| 建議 DCA | 可投資現金流 × 80% |
| 積極 DCA | 可投資現金流 × 100%，但不得低於安全水位 |

## 12. 情境與 Monte Carlo 規則

### 12.1 內建情境

至少保留：

- AI 超級大牛市
- 中性成長 / 估值正常化
- 先殺估值，再恢復
- AI 成長是真的，但股價表現不如預期
- 景氣壓力 / 收入衰退 / 家庭支出壓力疊加
- 自訂情境

### 12.2 Monte Carlo 必須使用共用 shock tape

為了公平比較不同組合，同一個 simulation_id 必須使用同一組市場環境。

```text
simulation_id = 1
同一組 market shock
同一組 risk_group shock
同一年份 shock
套用到 Current / Recommended / Custom / Candidate / VOO
```

否則勝率比較會失真。

| 指標 | 重要性 |

| P5 worst case | 極高 |
| 資產耗盡機率 | 極高 |
| 最大回撤 | 極高 |

| 達標 5,000 萬機率 | 高 |

## 18. 專案北極星

這個系統的最終目的不是讓 Ben 每天盯盤，也不是追逐最高報酬。

它的北極星是：

> **幫 Ben 用資料看清楚自己的財務現況，規劃通往富足與自由的人生路線，並在家庭責任、教育費、事業現金流衰退、房貸槓桿與股票波動之間，找到長期勝率最高、最不容易中途出局的資產配置。**
